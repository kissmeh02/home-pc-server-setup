100g upgrade path
The Mikrotik CRS317 has one 100G QSFP28 port today. When you're ready for full 100G, add a Mellanox ConnectX-5 100G NIC to PC 1 and NAS (~$150 each used), connect them to the QSFP28 port via a 100G DAC breakout cable, and your storage traffic runs at 100 Gbps. The rest of the network stays at 10/25G. Full upgrade cost: ~$400.
Current 25G backbone between PC 1 and NAS gives ~3 GB/s real-world throughput — enough for simultaneous 4K transcoding, NFS mounts, and backup jobs with no bottleneck. 100G (12.5 GB/s) is overkill today but the hardware is ready.

frigate detection pipeline
1
Camera → RTSP
8 cameras stream H.264/H.265 over VLAN 50 to Frigate VM. No internet. Local only.
2
GPU decodes
RTX 4070 #2 NVDEC decodes all 8 streams in hardware. Zero CPU load. Real-time.
3
Coral infers
Decoded frames to Coral TPU. Object detection at 100+ FPS. Under 50ms per frame.
4
Event fires
Person detected → Frigate sends event to HA via MQTT. HA automation runs.
5
Notification
HA sends push notification with snapshot image to your phone within 2–3 seconds.
6
Clip saved
10-second event clip saved to 12 TB dedicated IronWolf Pro. 90+ days retention.
Entire system runs with zero cloud subscription. No Ring, no Nest, no ADT monthly fee. Equivalent subscription security would cost $50–80/month. This system pays for itself in 18–24 months.
Camera VLAN 50 is strictly isolated — cameras can only reach the Frigate VM IP, nothing else. A compromised camera cannot reach the internet, your network devices, or any other service. This is the correct way to deploy IP cameras.
Grow room camera is on VLAN 40 (grow), not VLAN 50 (cameras). It streams to both Frigate VM (for recording) and grow HA VM (for timelapse/monitoring). pfSense firewall rules handle the selective access.

proxmox ve — pc 1 + pc 2 setup
1
Flash and boot Proxmox installer
Download Proxmox VE 8 ISO from proxmox.com. Flash to USB with Balena Etcher. Boot from USB. Install to dedicated NVMe boot drive. Set static IP during install.
Target disk: /dev/nvme0n1 (256 GB boot NVMe) IP: 192.168.10.10 (PC 1) / 192.168.10.11 (PC 2) Gateway: 192.168.10.1 (pfSense) DNS: 192.168.10.10 (Pi-hole, set after)
2
Enable IOMMU (PC 1 only — for GPU passthrough)
Edit GRUB and kernel modules to enable IOMMU passthrough. Required before creating GPU-passthrough VMs.
# Edit /etc/default/grub GRUB_CMDLINE_LINUX_DEFAULT="quiet amd_iommu=on iommu=pt" # Update grub and add kernel modules update-grub echo "vfio vfio_iommu_type1 vfio_pci vfio_virqfd" >> /etc/modules # Find GPU PCI IDs lspci -nn | grep NVIDIA # Add both 4070 PCI IDs to /etc/modprobe.d/vfio.conf echo "options vfio-pci ids=10de:2786,10de:22bc" > /etc/modprobe.d/vfio.conf update-initramfs -u && reboot
3
Configure ZFS storage pools
Create separate ZFS pools for VM disks (on NVMe) and HDD data. Use Proxmox web UI under Datacenter → Storage.
# Create VM storage pool on NVMe (done in Proxmox UI) # Pool name: vm-storage, Type: ZFS, Device: NVMe # Create media mirror pool (PC 1 only, 2× HDD) zpool create media mirror /dev/sda /dev/sdb zfs set compression=lz4 media zfs set atime=off media # Create security NVR pool (PC 1 only, 1× HDD) zpool create nvr-footage /dev/sdc zfs set compression=lz4 nvr-footage
4
Create VMs and LXC containers
Use Proxmox web UI (https://192.168.10.10:8006). Create each VM from Ubuntu 22.04 ISO. For LXCs, use Proxmox CT templates. Assign vCPU, RAM, and disk as per VM layout table.
# Key settings per VM: # VM 1 Plex: 4 vCPU, 16GB RAM, 300GB disk, add RTX 4070 #1 PCI device # VM 6 Frigate: 4 vCPU, 32GB RAM, 500GB disk, add RTX 4070 #2 + USB Coral TPU # All VMs: VirtIO NIC, VirtIO disk, CPU type = host
5
Configure Proxmox Backup Server
Install Proxmox Backup Server (PBS) as an LXC on PC 2. Configure backup jobs to run nightly at 2 AM. Point backup target to NAS via NFS.
# In Proxmox UI: Datacenter → Backup # Add schedule: daily at 02:00 # Mode: Snapshot (zero downtime) # Retention: keep-daily=7, keep-weekly=4, keep-monthly=3
truenas scale — nas setup
1
Install TrueNAS SCALE
Download TrueNAS SCALE ISO. Flash to USB. Boot NAS. Install to 256 GB boot NVMe. Access web UI at the NAS IP after reboot. Configure network interface with static IP.
NAS IP: 192.168.10.12 Web UI: https://192.168.10.12
2
Create RAIDZ2 storage pool
In TrueNAS UI: Storage → Create Pool. Select all 6× 20 TB HDDs. Choose RAIDZ2. Name the pool "data". Enable LZ4 compression. Set record size to 1M for media workloads.
Pool name: data Layout: RAIDZ2 (6 wide) Disks: 6× 20TB IronWolf Pro Compression: LZ4 Record size: 1M (media) / 128K (general)
3
Create datasets and NFS shares
Create datasets for media, backups, and company data. Enable NFS shares. PC 1 VMs mount the media share for Plex library. All PCs backup to the backups dataset.
Datasets: data/media → NFS share → PC 1 VM 1 Plex data/backups → NFS share → Proxmox Backup Server data/company → NFS share → PC 2 VM 2/3 data/personal → NFS share → PC 1 VM 3 NFS: enable v4.1, set network to VLAN 10 only
4
Configure automated snapshots + replication
TrueNAS has built-in snapshot scheduling. Set hourly snapshots with 7-day retention. Configure cloud sync task to replicate to Backblaze B2 for offsite backup.
Snapshot task: every hour, keep 168 (7 days) Cloud sync: data/backups → Backblaze B2 Schedule: nightly 03:00 Mode: Push + Mirror
5
Enable MinIO S3 endpoint
In TrueNAS SCALE, deploy MinIO as an app (Apps → MinIO). Point storage to the data/s3 dataset. This gives all your services an S3-compatible local endpoint for object storage and backups.
MinIO endpoint: http://192.168.10.12:9000 Access key: (set in MinIO console) Bucket: 6phene-personal / 6phene-company
pfsense firewall — home network
1
Initial pfSense setup
Connect Protectli WAN port to modem. Connect LAN port to CRS317 switch. Access pfSense web UI at 192.168.1.1 on first boot. Run setup wizard. Set LAN IP to 192.168.10.1.
2
Create VLANs
In pfSense: Interfaces → Assignments → VLANs. Create VLAN 10 (servers), 20 (desktop), 30 (IoT), 40 (grow), 50 (cameras), 60 (guest). Assign each a pfSense interface with its own subnet.
VLAN 10: 192.168.10.0/24 (servers) VLAN 20: 192.168.20.0/24 (desktop) VLAN 30: 192.168.30.0/24 (IoT) VLAN 40: 192.168.40.0/24 (grow) VLAN 50: 192.168.50.0/24 (cameras) VLAN 60: 192.168.60.0/24 (guest)
3
Firewall rules (critical for security)
Block inter-VLAN traffic by default. Add specific allow rules only where needed. Cameras must only reach Frigate VM. IoT must only reach HA VM.
VLAN 50 (cameras) rules: ALLOW: dst 192.168.10.X port 1935,8554 (Frigate RTSP) BLOCK: all other traffic VLAN 30 (IoT) rules: ALLOW: dst 192.168.10.X port 8123 (HA) BLOCK: all other traffic VLAN 60 (guest): ALLOW: dst WAN (internet only) BLOCK: all RFC1918 (no LAN access)
4
Configure Pi-hole as DHCP DNS
In pfSense DHCP server settings for each VLAN, set DNS server to PC 1 Pi-hole IP (192.168.10.X). All devices automatically use Pi-hole for DNS without any per-device configuration.
tailscale — secure remote access
1
Install Tailscale on all machines
Install on PC 1, PC 2 dev VM, PC 3, and NAS. Each machine gets a Tailscale IP in the 100.x.x.x range. They can communicate securely from anywhere without port forwarding.
# Ubuntu/Debian (PC 1, PC 2 VMs, NAS) curl -fsSL https://tailscale.com/install.sh | sh sudo tailscale up # Windows (PC 3) # Download from tailscale.com/download

Set PC 2 dev VM as subnet router
This lets you access ALL home LAN devices (192.168.10.0/24) from PC 3 when you're away, not just Tailscale-enrolled machines. One command.

Configure SSH from PC 3 to PC 2
On PC 3 (WSL2 Ubuntu): generate SSH key pair. Copy public key to PC 2 dev VM. Add VS Code Remote SSH config for seamless development access.

frigate nvr setup — vm 6
1
Install Frigate in VM 6
VM 6 has RTX 4070 #2 passed through and Coral TPU via USB. Install Docker, deploy Frigate container with GPU + TPU environment variables.
docker run -d \ --name frigate \ --privileged \ --shm-size=512m \ -e FRIGATE_RTSP_PASSWORD="yourpass" \ --device /dev/apex_0 \ # Coral TPU -v /dev/dri:/dev/dri \ # GPU decode -v /config:/config \ -v /nvr-footage:/media/frigate \ -p 5000:5000 \ ghcr.io/blakeblackshear/frigate:stable
2
Configure cameras in frigate config
Add each camera's RTSP URL to frigate config.yml. Enable Coral TPU as detector. Enable NVDEC for GPU stream decoding. Set recording and snapshot retention.
detectors: coral: type: edgetpu device: usb cameras: front_door: ffmpeg: hwaccel_args: preset-nvidia-h264 inputs: - path: rtsp://192.168.50.10:554/stream1 detect: {width: 3840, height: 2160, fps: 5} record: enabled: true retain: {days: 90, mode: motion}
3
Connect Frigate to Home Assistant
Install Frigate integration in HA. Configure MQTT broker (Mosquitto add-on in HA). Frigate sends detection events via MQTT → HA triggers automations → push notifications with camera snapshot.
# In HA: Settings → Integrations → Add → Frigate # Frigate URL: http://192.168.10.X:5000 # Configure notification automation: trigger: platform: mqtt topic: frigate/events action: service: notify.mobile_app data: message: "Person detected: {{ trigger.payload_json.camera }}" data: image: "/api/frigate/notifications/{{trigger.payload_json.id}}/thumbnail.jpg"
