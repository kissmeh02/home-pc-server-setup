Home NAS Server
home · TrueNAS SCALE · ZFS RAIDZ2 · S3-compatible · backup target · optional Plex offload
why add a dedicated nas
PC 1 runs Proxmox — its storage is optimised for VM disks. A dedicated NAS running TrueNAS SCALE gives you a purpose-built ZFS storage pool with S3-compatible object storage (MinIO built-in), automated snapshots, ZFS send/receive replication, and a proper backup target for all three PCs. It also offloads Plex library storage from PC 1's HDDs, freeing PC 1 for pure compute.
TrueNAS SCALE (Linux-based) supports running apps in Docker/K3s alongside the NAS functions — you can run Plex Media Server directly on the NAS and free VM 1 on PC 1 entirely if you want, or use the NAS purely as storage.

architecture notes
RAIDZ2 with 6× 20 TB is the sweet spot — 2-drive fault tolerance means you can lose any two drives simultaneously and still be running. Rebuild time on a single 20 TB drive is ~12 hours. During rebuild, RAIDZ2 stays online and accessible.
ECC RAM is non-negotiable for ZFS. Without ECC, a single bit flip in RAM can silently corrupt data before it hits the checksummed filesystem. The W680-ACE supports ECC unbuffered DDR5 which is rare on consumer-accessible boards.
The NVMe cache drive serves dual duty: the first partition is the ZFS SLOG (sync write journal — dramatically speeds up NFS writes from PC 1), and the second is L2ARC (a hot read cache that keeps frequently accessed data in NVMe speed).
NAS sits on the 100G core switch via 25G SFP28 DAC cable. PC 1 connects at 25G. This means NFS mounts from PC 1 VMs to the NAS are effectively 25 Gbps — fast enough for real-time 4K Plex streaming with no buffering.
